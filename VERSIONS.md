# Versions

A running log of changes. Newest at the top. One line per change.

- **0.0.17** — Visual redesign toward a calmer Morgen / Routine feel. Body gradient swapped from grayscale + radial accents to a single linear blue → lilac wash (`#eef2ff → #f1eefe → #f7eefd → #fdf4ff`). Accent moved from Apple blue (`#0a84ff`) to indigo (`#6366f1`) with matching 50/100/600/700 tones so the brand colour harmonises with the new background. All glass surfaces removed: every `bg-white/X` + `backdrop-blur-*` pair replaced with solid `bg-white` (or lightly translucent `bg-white/95` to let the gradient peek through), and thin `border-slate-200/80` lines now do the heavy lifting that shadows used to. `shadow-card` and `shadow-elevated` are gone from active states, headers, footers, and content cards; `shadow-panel` survives only on the task detail panel and the sign-in card, redefined in `tailwind.config.ts` to a much softer `0 10px 30px -12px rgba(31,35,48,0.18)`. Sidebar active rows and segmented filter "selected" pills now use a tinted `bg-accent-50 text-accent-700` instead of a white card. Today column drops the gradient-and-accent-stripe combo for a simple `bg-accent-50/40` tint. Hover backgrounds standardised from `stone-*` to `slate-*` so the cool palette stays consistent.
- **0.0.16** — Mobile responsiveness pass. `useDayCount` gained a third tier (3 days < 640px, 4 days 640–1279px, 7 days ≥ 1280px) and the calendar grid now stacks vertically on phones (`flex-col sm:flex-row`) with per-row dividers, while min-height only applies on `sm+` so each stacked column sizes to content. The persistent sidebar becomes an off-canvas drawer on `< md`: `fixed inset-y-0 left-0 w-72` with backdrop and translate transform, opened by a hamburger in a new mobile-only top bar that also shows the current view label. The drawer auto-closes when the active view changes. `TaskDetailPanel` slides up from the bottom on phones (`inset-x-0 bottom-0 top-12`, translate-y) and keeps the right-side panel layout on `sm+`; padding tightened to 4px/5px on small screens. `ProjectView` / `FocusView` / `CustomListView` outer padding reduced to `px-4 py-5` on mobile, headings drop from 26 → 22px, project quick-add stretches full-width and segmented filter wraps above it on phones, and row paddings drop from `px-5` to `px-4`.
- **0.0.15** — Wishlist item 7: Focus view. New top-level sidebar entry directly below "Calendar" routes to `FocusView`, a single-column page with three grouped sections (Overdue / Due today / Scheduled for today) plus a quick-add at the top that creates a task with `scheduled_date = today`. `useFocusTasks` fetches uncompleted tasks where `scheduled_date = today OR due_date <= today`, applies the same `start_date <= scheduled_date` visibility rule the calendar uses, and partitions client-side. Reuses `TaskCard` so urgency colouring, bold-on-due, and the recurring glyph stay in sync. Empty sections hide their heading; an entirely empty view shows a single "Inbox zero" state. View union extended with `{ kind: "focus" }`.
- **0.0.14** — Calendar visibility fix for future-start tasks: `useTasks` previously hid any task whose `start_date` was later than today, which meant recurring instances (start_date = scheduled_date = some future Monday) never appeared in their own future column. Now the rule is per-row: hide only when `start_date > scheduled_date` (the "delayed start" one-off case). Filter is applied client-side because PostgREST can't compare two columns in a URL filter.
- **0.0.13** — Recurrence reworked from scheduled-date-anchored (with offsets) to per-field. Migration `0004_recurrence_per_field.sql` wipes existing templates, drops `rrule`/`dtstart`/`start_offset_days`/`due_offset_days`, and adds `start_rrule`/`start_dtstart`/`due_rrule`/`due_dtstart` plus a CHECK requiring at least one pair to be populated. `RecurrenceEditor` now renders two independent sections — one for the start date, one for the due date — each disabled until the matching date is set on the task. The generator picks a "driver" rule per template: if `start_rrule` is set, every start occurrence in the 60-day window spawns one instance (`scheduled_date = start_date = occurrence`, `due_date = next due-rule occurrence on or after start_date`); otherwise the due rule drives (`scheduled_date = due_date = occurrence`, `start_date = null`). `lib/recurrence.ts` gains `nextOccurrenceOnOrAfter` and drops the offset helpers.
- **0.0.12** — Wishlist items 1, 2, 3 landed. (1) Overdue and due-today task titles render in `font-semibold` everywhere a task appears, via a new `isDueOrOverdue()` helper in `lib/dates.ts`. (2) `ProjectView` gets an inline `ProjectQuickAdd` that schedules new tasks for today and assigns them to the open project; `useCreateTask` accepts an optional `project_id`. (3) Recurring tasks via the template-plus-instances model: new migration `0003_recurrence.sql` adds `task_templates` (rrule, dtstart, start_offset_days, due_offset_days, owner-scoped RLS) and a nullable `tasks.template_id` FK with `on delete set null`. `rrule` lib parses RFC 5545; new `lib/recurrence.ts` exposes `buildRRule` / `parseRRule` / `occurrencesBetween` / `applyOffset` with UTC-only date arithmetic to avoid timezone drift. `useTaskTemplates` hooks cover CRUD; `useRecurrenceGenerator` runs once per app load, plus `runRecurrenceGenerator(qc)` is invoked from the panel right after a recurrence edit so future instances appear immediately. New `RecurrenceEditor` lives in the task detail panel: presets (Daily / Weekly on <weekday> / Monthly on day N / Custom every N day–week–month) plus ends (never / after N / on date), with a Save/Cancel confirmation step. A `↻` glyph on `TaskCard` flags recurring instances.
- **0.0.11** — Expanded `WISHLIST.md` from a 6-line checklist into actionable specs: for each item (bold overdue tasks, project-view quick-add, recurring dates, calendar drag-and-drop, daily Discord digest, calendar overlay) wrote out the goal, design decisions, implementation notes, and a manual test plan, plus an S/M/L size estimate. Linked from `CLAUDE.md` as the post-M7.5 backlog.
- **0.0.10** — Milestone 7.5: auth + RLS (prerequisite for the public deploy). Supabase magic-link sign-in, session-gated app, sign-out + user chip in the sidebar footer. `AuthProvider` subscribes to `onAuthStateChange`; `SignIn` posts an OTP with `emailRedirectTo` set to the current origin so the round-trip lands back on the app. Migration `0002_auth.sql` wipes all existing rows, adds `user_id uuid not null default auth.uid()` to `projects` / `tasks` / `custom_lists` / `custom_list_items`, enables RLS, and adds one policy per command scoped to `auth.uid() = user_id`. Existing client hooks don't need user_id explicitly — the column default fills it on insert and RLS filters reads.
- **0.0.9** — Milestone 7: polish & deploy. GitHub Actions workflow deploys to GitHub Pages on every push to `main`, with `VITE_SUPABASE_URL`/`VITE_SUPABASE_ANON_KEY` injected from repo secrets and Vite `base` switched to `/NextDays/` under Actions. Sweeping macOS-style visual pass: Apple-gray canvas with soft radial accent gradients, glass sidebar (`backdrop-blur-xl`), floating calendar card, today column with subtle gradient + accent stripe, circular accent-coloured checkboxes, segmented filter pills, glass task-detail panel with hero-sized title, refined empty states, custom slim scrollbars, accent focus rings, refined typography (Inter, tighter tracking on display). Tailwind theme extended with `shadow-card`/`elevated`/`panel`, `accent` colour, `ease-out-soft` curve, and `animate-fade-up`.
- **0.0.8** — Milestone 6: custom lists. Sidebar gains a "Lists" section below Projects with inline create / rename / delete (`ListNameForm`). Clicking a list opens `CustomListView` — active/completed/all filter, inline "+ Add item" input, checkbox toggle, edit-title-on-blur, expand-to-show notes, and per-item delete. `View` state extended with `{ kind: "list"; id }`. Hooks: `useCustomLists` (list CRUD) and `useCustomListItems` (item CRUD). Items stay isolated from tasks per spec §5.10.
- **0.0.7** — Milestone 5: projects. Left sidebar lists projects with coloured dots; "+ New project" and per-row Edit/Delete reveal an inline `ProjectForm` (name, 10-colour palette, Personal/Work toggle). Clicking a project opens `ProjectView` — flat task list filterable by active/completed/all, opens the same `TaskDetailPanel` on click. Task cards now show a small dot in the project's colour. View state lives in a tiny `ViewProvider`. Interface font switched to Inter (variable, via `@fontsource-variable/inter`) and wired through Tailwind's `font-sans`.
- **0.0.6** — Milestone 4: click a task → slide-in `TaskDetailPanel` from the right with editable title, notes, start/due dates, project (dropdown of existing projects, empty until M5), and comma-separated tags. Each field auto-saves on blur via `useUpdateTask` partial-patch. Panel closes on Escape, backdrop click, or × button. Selection state lives in a tiny `SelectionProvider` context.
- **0.0.5** — Milestone 3: task interactions. Per-column quick-add (Enter creates task with `scheduled_date` = column date), functional checkbox toggle (sets `completed`/`completed_at`, re-sorts to bottom of column), client-side rollover on app load — unchecked tasks with `scheduled_date < today` move to today, with work-project weekend exception per §5.3.
- **0.0.4** — Milestone 2: read-only `CalendarStrip` with responsive 4/7-day window (`useDayCount`), today highlighted, tasks fetched via `useTasks` and sorted per §5.2 (due-soon pinned, completed greyed/struck at bottom), due-date 🔔 indicators per §5.4, future `start_date` tasks hidden. SmokeScreen removed.

The standalone `@vitejs/plugin-react-oxc` package is deprecated; its functionality is now in `@vitejs/plugin-react`. Typecheck + build green, dev server starts warning-free.
